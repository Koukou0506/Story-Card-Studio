# Story Card Studio 页面蓝图（Phase C2.0）

## 1. 三种视觉方向

### 方向 A：Greenboard 式宽松编辑工作台

**视觉特点**

- 40–56px 大标题；
- 大面积暖米白画布和 24px 圆角块面；
- 页面区块间距 64–80px；
- 每屏只突出一个主题和一个主 CTA；
- 卡片数量少，依靠抽屉和分步展开容纳复杂内容。

**适合页面**

- 项目首页；
- 创意输入；
- 新建向导；
- 角色卡基础编辑；
- 空状态和首次使用流程。

**信息密度**：低到中。

**开发成本**：高。现有复杂工作区需要重新分组、隐藏低频信息并建立抽屉/分步结构。

**可维护性**：中等。简单页面清楚，但高密度数据页容易不断增加例外。

**移动端适配难度**：中等。块面易降为单列，但大标题和大留白需要重新收敛。

**风险**

- 世界书、规划和连续性的信息吞吐量下降；
- 频繁展开/收起会增加操作次数；
- 如果只模仿营销页比例，会牺牲专业工具效率；
- 参考站的大图轮播不适合本应用，不能作为复杂数据的主要容器。

### 方向 B：紧凑专业创作工具

**视觉特点**

- 固定左侧导航、密集工具条、34–40px 控件；
- 表格、树形列表和三栏主从布局为主；
- 16–24px 标题，页面留白较少；
- 版本、来源、问题和状态一直可见；
- 使用更多分隔线和粘性工具栏。

**适合页面**

- 世界书；
- 小说规划；
- 连续性中心；
- 导入导出；
- 版本、来源和问题管理。

**信息密度**：高。

**开发成本**：中等。与现有功能堆叠方式接近，但需要共享表格、工具条和检查器。

**可维护性**：较高。复杂数据有稳定模式，前提是组件和 Token 统一。

**移动端适配难度**：高。三栏、表格和粘性工具条必须转换为主从导航和抽屉。

**风险**

- 创意输入和正文写作显得压迫；
- 新用户学习成本高；
- 容易回到“所有功能同时出现”的现状；
- 小字号和触控目标可能重新产生可访问性问题。

### 方向 C：分层密度的 Editorial Creative Workspace

**视觉特点**

- 外壳、页面头部、首页和正文采用宽松 Editorial 节奏；
- 数据列表、来源、版本和问题采用 Compact 密度；
- 同一页面通过“列表—编辑器—检查器”分配信息，而不是无限向下堆叠；
- 40px H1、16px 主正文、17px 正文编辑字号；
- 大块面用 16–24px 圆角，数据控件用 8–10px 圆角；
- 钴蓝只用于主操作、选中、链接和焦点。

**适合页面**

- 全部现有工作区；
- 特别适合角色卡、世界书和正文写作；
- 连续性和规划可在同一系统中切换到 Compact 密度。

**信息密度**：分层；导航和编辑舒适，工具与数据紧凑。

**开发成本**：中高。需要先建立 App Shell、共享组件和两套密度 Token，再逐页迁移。

**可维护性**：最高。页面复用同一视觉语法，差异只来自密度和布局槽位。

**移动端适配难度**：中等。三栏可统一降级为列表 + 主区 + 抽屉，C2.1 不必为每页重新发明模式。

**风险**

- 如果没有明确密度边界，同页可能看起来像两套系统；
- 初期需要严格治理共享组件，不能让页面继续硬编码；
- 迁移期间新旧页面会短期共存，需要固定迁移顺序。

### 推荐

推荐方向 C。

理由：当前产品同时包含创意、长表单、长文写作、结构化数据、版本和连续性检查。单一低密度或高密度方案都会牺牲一半场景。方向 C 可以吸收 Greenboard 的留白、标题比例、粘性导航和单一主 CTA，同时为世界书、规划和连续性保留专业工具效率。

推荐默认：

- 全局为 Comfortable；
- 世界书列表、来源、版本、问题和连续性表格使用 Compact；
- 用户可在设置中选择默认密度，但页面级关键密度由设计系统控制；
- 第一批只迁移外壳、首页、角色卡、世界书和正文写作；
- 第二批再迁移分析、规划、连续性、导入导出和设置。

## 2. 共用页面语法

所有一级页面遵循：

```text
App Shell
├─ 左侧主导航（工作流分组）
├─ 上下文顶栏（项目、保存状态、搜索、页面操作）
└─ 主内容
   ├─ Page Header（H1、说明、状态、唯一主 CTA）
   ├─ 页面级警告 / 迁移提示
   └─ Workspace
      ├─ 列表或局部导航
      ├─ 主编辑/报告区
      └─ 可收起检查器
```

层级规则：

1. 一级导航回答“我在哪个工作阶段”；
2. Page Header 回答“我正在处理哪个对象和版本”；
3. 局部导航回答“这个对象的哪一部分”；
4. 检查器回答“生成、来源、质量、版本和建议”；
5. 低频导入导出、删除和高级格式进入溢出菜单或对话框。

## 3. 应用外壳

### 页面目标

让用户始终知道当前项目、工作阶段、保存状态和可恢复操作，并为 A1–C1 所有工作区提供稳定导航。

### 信息优先级

1. 当前项目与当前工作区；
2. 本地保存/迁移状态；
3. 页面主操作；
4. 全局搜索和跨模块跳转；
5. 导入导出、设置和危险操作。

### 页面区域

```text
┌──────────────┬─────────────────────────────────────────────────────┐
│ 项目/品牌     │ 面包屑 · 页面标题       保存状态  搜索  主操作      │
├──────────────┼─────────────────────────────────────────────────────┤
│ 首页          │                                                     │
│ 创作          │                                                     │
│  创意输入     │                   主内容区                          │
│  角色卡       │                                                     │
│  世界书       │                                                     │
│ 规划          │                                                     │
│  剧情分析     │                                                     │
│  小说规划     │                                                     │
│ 写作          │                                                     │
│  正文写作     │                                                     │
│ 维护          │                                                     │
│  连续性中心   │                                                     │
├──────────────┤                                                     │
│ 导入导出 设置 │                                                     │
└──────────────┴─────────────────────────────────────────────────────┘
```

### 导航

- 左侧 240px，按“创作、规划、写作、维护”分组；
- 首页独立于工作流分组；
- 导入导出和设置固定在底部；
- 当前项使用 accent 左标记、浅底和 `aria-current="page"`；
- 不使用 Emoji 作为正式导航图标；
- 紧凑桌面可收为 80px，但每个图标必须有 Tooltip 和可访问名称。

### 主要组件

- Project Switcher；
- Grouped Navigation；
- Context Bar；
- Save Status；
- Global Search / Command Trigger；
- Page Action Slot；
- Migration / Recovery Banner。

### 次要组件

- 数据版本；
- 最近保存时间；
- Mock / 实际 Provider 状态；
- 帮助与方法文档入口；
- 用户密度和动效偏好。

### 空状态

没有本地项目时，外壳仍显示，但主内容进入项目首页空状态；导航中依赖项目的功能保持可见并解释为什么暂不可用，不直接消失。

### 加载状态

- 外壳和导航立即显示；
- 项目名称、保存状态和主内容使用骨架；
- 不使用全屏遮罩；
- localStorage 迁移期间禁止危险操作。

### 错误状态

- 迁移失败在顶栏下显示持久 Banner；
- 提供“导出原始恢复数据”和“查看错误说明”；
- 页面加载失败保留导航和项目切换；
- Toast 仅补充即时结果，不代替可恢复错误。

### 关键交互

- 切换一级页面后，焦点进入 Page H1；
- 未保存的临时草稿切换页面时显示范围明确的确认；
- 全局搜索可跳到角色、条目、章节、场景和 Canon 来源；
- 清除项目从 Header 移到设置危险区；
- 当前页主操作由页面注入，外壳不硬编码业务。

### 响应式变化

- ≥1280px：240px 固定导航；
- 1024–1279px：80px 收窄导航，检查器默认可收起；
- 768–1023px：导航为左侧抽屉，顶栏保留项目和页面操作；
- <768px：C2.1 完整设计，本轮确保抽屉和主内容不会互相遮挡。

### 应删除或合并的现有元素

- 删除顶部 9 个 Emoji 标签条；
- 合并品牌标题、Phase 副标题和数据版本为 Project Switcher + About 信息；
- 将“角色卡质量检查”移入角色卡；
- 将“清除数据”移入设置危险区；
- 把分散 Toast 和 migration notice 统一为共享反馈组件。

## 4. 项目首页

### 页面目标

为新项目提供清楚的开始路径，为已有项目提供“继续创作”和项目健康总览。

### 信息优先级

1. 继续最近工作；
2. 下一写作目标和关键继承状态；
3. 项目进度与严重风险；
4. 最近角色、世界书、规划和正文；
5. 快速新建或导入。

### 页面区域

```text
Page Header：项目名 / 最近保存 / 项目设置

Hero：一句项目摘要
      [继续写作]  [查看下一章上下文]

进度条带：规划 / 初稿 / 修订 / 正式字数

两列区域
├─ 最近工作（场景、章节、角色、世界书）
└─ 需要处理（严重问题、Canon 冲突、过期摘要）

快速开始：创意输入 / 导入角色卡 / 创建世界书 / 建立规划
```

### 导航

- Page Header 的项目名可进入设置；
- “继续写作”跳到最近 accepted 场景；
- 健康项直接跳到对应 Continuity Issue；
- 快速开始不复制一级导航，只呈现有上下文的下一步。

### 主要组件

- Continue Card；
- Progress Strip；
- Recent Work List；
- Attention Queue；
- Next Chapter Summary；
- Quick Start Cards。

### 次要组件

- 数据版本；
- 最近编辑时间；
- 当前采用规划和正文版本；
- Mock Provider 提示；
- 本地存储说明。

### 空状态

新项目显示：

- H1“开始你的创作项目”；
- 原始创意输入的短说明；
- 主 CTA“写下创意”；
- 次操作“导入角色卡/世界书/项目 JSON”；
- 明确说明数据保存在本地，Mock 无需密钥。

不在空状态一次展示所有 A1–C1 功能。

### 加载状态

- Hero 先显示项目名骨架；
- Progress 和最近工作分别骨架化；
- 局部失败不阻断 Continue Card。

### 错误状态

- 无 accepted 正文时，“继续写作”降级为“打开章节规划”；
- 进度计算失败时显示“数据仍安全保存”，提供重新计算；
- 迁移失败沿用全局恢复 Banner。

### 关键交互

- Continue Card 记住最近对象和编辑位置；
- Attention Queue 支持按严重程度折叠，但不在首页直接修复；
- Progress Strip 点击进入连续性中心的写作进度；
- 首页不会调用模型，不会自动刷新消耗 API。

### 响应式变化

- 桌面：Hero 和进度全宽，最近工作/需要处理两列；
- 平板：两列降为单列，Continue Card 始终第一；
- 小屏基础：快速开始横向卡片改为列表。

### 应删除或合并的现有元素

- 不再把“创意输入 + GenerationPanel + CharacterEditor”当作首页；
- 不在首页重复完整角色卡表单；
- 不把所有导入导出格式铺在首页；
- 项目健康只展示摘要，不复制连续性中心全部统计。

## 5. 角色卡编辑

### 页面目标

高效编辑当前角色卡，同时随时看到质量、来源、版本和关联 Character Book，不让生成面板长期占据主空间。

### 信息优先级

1. 角色名称、描述、人格和互动内容；
2. 保存状态和当前版本；
3. 质量问题与缺失字段；
4. 高级提示、元数据和 extensions；
5. 重新生成、导入导出和危险操作。

### 页面区域

```text
Page Header：角色卡 / 角色选择 / 版本 / [生成或更新]

┌──────────────┬───────────────────────────────┬──────────────────┐
│ 角色与版本    │ 编辑表单                       │ 检查器            │
│ 搜索          │ 基础 / 人格 / 互动 / 提示 / 元数据│ 质量 / 来源 / 格式 │
│ 角色列表      │ 720–840px 可读编辑列            │ 关联世界书         │
└──────────────┴───────────────────────────────┴──────────────────┘
```

### 导航

- 左侧选择角色和版本；第一版即使只有一个角色，也保留稳定结构；
- 中部使用章节导航，不超过 5 项：基础、人格、互动、提示、元数据；
- 右侧检查器使用质量、来源、格式三组；
- Character Book 入口位于“格式”而不是独立长卡片。

### 主要组件

- Character List / Version Switcher；
- Field Section；
- Sticky Save Status；
- Quality Inspector；
- Source / Extension Inspector；
- Generate Dialog。

### 次要组件

- Tag editor；
- Greeting list editor；
- 字数和 Token 估算；
- 关联世界书；
- 导入导出格式摘要。

### 空状态

- 无角色卡：主 CTA“从创意生成角色卡”；
- 次操作“新建空白角色卡”“导入 Character Card V2”；
- 如果已有原始创意，显示一行摘要但不自动生成。

### 加载状态

- 角色列表和表单分别骨架化；
- 生成中的结果进入临时预览，当前字段保持可读；
- 禁止用整页 Spinner 覆盖已有角色内容。

### 错误状态

- 字段校验显示在字段下方并关联输入；
- Provider 错误在生成对话框内显示；
- extensions 解析问题在格式检查器显示，正文编辑仍可继续；
- 保存失败持续显示恢复/导出入口。

### 关键交互

- 自动保存显示“保存中 / 已保存 / 失败”；
- 质量问题点击后聚焦对应字段；
- 重新生成默认创建预览，不直接覆盖；
- 导入检测到当前角色时先预览替换范围；
- extensions 只读展示并明确 round-trip 保护；
- 长文本支持独立扩大编辑。

### 响应式变化

- ≥1280px：260px 列表 + 主编辑 + 320px 检查器；
- 1024–1279px：角色列表收窄，检查器为可收起面板；
- 768–1023px：角色列表为抽屉，检查器为底部 Sheet，编辑器单列；
- 高级字段从两列降为一列。

### 应删除或合并的现有元素

- 删除固定占左列的 Generation Panel；
- 合并连续长表单为五个字段章节；
- 把角色质量从一级“项目质量”迁回检查器；
- 合并 extensions 提示、版本和格式信息；
- 不再用 Emoji 作为 Card Header 图标。

## 6. 世界书编辑

### 页面目标

在大量条目中快速定位、筛选和批量管理，同时保持单条编辑、格式兼容、模拟和安全写回可见。

### 信息优先级

1. 当前世界书和条目；
2. 条目启用、关键词、正文和常驻状态；
3. 筛选、批量操作与兼容性警告；
4. 激活模拟和质量；
5. 生成、导入导出、Character Book 写回。

### 页面区域

```text
Page Header：世界书 / 来源格式 / 统计 / [新增条目]

┌──────────────┬───────────────────────────────┬──────────────────┐
│ 世界书列表    │ 条目列表 / 表格                 │ 条目检查器        │
│ 新建/导入     │ 搜索 分类 状态 批量操作          │ 基础 / 高级        │
│ 书籍元数据    │ 选中条目                        │ 质量 / 模拟 / 格式 │
└──────────────┴───────────────────────────────┴──────────────────┘
```

### 导航

- 左侧在世界书间切换；
- 中部支持“舒适卡片 / 紧凑表格”视图；
- 右侧检查器分为编辑、质量、模拟、格式；
- 页面溢出菜单包含生成、导入、导出、复制和删除书籍。

### 主要组件

- Lorebook List；
- Entry Toolbar；
- Entry List / Data Table；
- Entry Inspector；
- Bulk Action Bar；
- Compatibility Warning；
- Activation Simulator；
- Character Book Write Dialog。

### 次要组件

- 世界书统计；
- 来源格式和最近修改；
- 未知字段计数；
- 估算正文长度；
- 本地近似模拟说明。

### 空状态

- 没有世界书：主 CTA“从当前项目生成世界书”；
- 次操作“新建空白世界书”“导入 JSON”“从角色卡读取 Character Book”；
- 说明三种入口的差异和 round-trip 保护。

### 加载状态

- 书籍和条目列表先显示骨架；
- 生成中的新书作为“未保存草稿”出现在列表顶部；
- 现有书籍和条目保持可浏览；
- 模拟与质量独立加载，不锁住编辑器。

### 错误状态

- 导入失败显示文件名、格式识别结果和可理解原因；
- 兼容性警告关联到具体字段；
- 写回冲突显示新增、修改、冲突和保留数量；
- Provider 错误只影响生成，不影响编辑和导出；
- 未知字段永远不因为正文编辑而消失。

### 关键交互

- 条目单击选中，双击或 Enter 打开编辑；
- 多选后出现批量启用/停用/分类/删除工具条；
- 排序使用按钮为基础，可在后续增加拖动；
- 模拟命中的条目可一键定位；
- 兼容性警告点击后定位高级字段；
- Character Book 写回必须在对话框中选择取消、替换或合并；
- 冲突绝不静默覆盖。

### 响应式变化

- ≥1280px：240px 书籍列表 + 主列表 + 360px 检查器；
- 1024–1279px：书籍列表收窄，检查器默认折叠；
- 768–1023px：先显示条目列表，书籍和检查器用抽屉；
- 紧凑表格在平板转为主从卡片，不横向滚动核心正文。

### 应删除或合并的现有元素

- 删除当前四段纵向长页面结构；
- 合并总览与书籍选择为左侧列表；
- 合并质量、模拟和格式为右侧检查器；
- 生成配置移入对话框，不常驻左栏；
- 导入导出和 Character Book 写回不再占用全宽 Card。

## 7. 正文写作

### 页面目标

让作者专注于当前场景正文，同时以最小干扰访问 Scene Plan、生成、质量、候选事实、Revision 和版本。

### 信息优先级

1. 当前正文和光标位置；
2. 当前章节/场景与保存状态；
3. Scene Plan 目标、视角、入口/出口状态；
4. 生成/局部修订和 Revision；
5. 质量、Coverage、候选事实和状态；
6. 导出、项目管理和低频设置。

### 页面区域

```text
Context Bar：卷 / 章 / 场景 / 版本 · 已保存 · 字数 · [生成]

┌──────────────┬─────────────────────────────────┬──────────────────┐
│ 章节/场景树   │ 760px 正文画布                   │ 写作检查器        │
│ 状态与进度    │ 段落锁定标记                     │ 计划 / 生成        │
│ 最近版本      │ 选区浮动工具条                   │ 检查 / 版本        │
└──────────────┴─────────────────────────────────┴──────────────────┘

可选底部：Revision Diff / 流式临时草稿
```

### 导航

- 左侧树按卷/章/场景分组，显示完成、未完成和版本状态；
- 右侧一级只保留四组：计划、生成、检查、版本；
- Style、Language、Context、Facts、States、Revisions 作为四组内的二级列表；
- 导入导出、删除正文项目和全局 Style Profile 进入页面菜单或设置。

### 主要组件

- Chapter / Scene Tree；
- Prose Canvas；
- Selection Toolbar；
- Paragraph Lock Marker；
- Generation Composer；
- Plan Inspector；
- Quality / Coverage Inspector；
- Revision Diff；
- Version History。

### 次要组件

- 字数、段落数、保存时间；
- Provider/Model；
- 前文预算和截断提示；
- Candidate Fact / State count；
- B2 偏离标记；
- Style Profile 与 Language Constraint 快速摘要。

### 空状态

- 没有 Manuscript：主 CTA“从当前 Scene Plan 创建正文项目”；
- 没有 B2：提示先打开小说规划创建场景，并提供直达按钮；
- 有 Manuscript 但场景无正文：主 CTA“生成完整场景初稿”，次操作“直接开始写作”。

### 加载状态

- 正文画布先显示 accepted 版本，不被加载遮盖；
- 流式输出进入独立临时草稿面板；
- 取消后保留 partial 内容并标记 incomplete；
- Coverage、事实、状态和质量作为生成后的独立分析步骤显示进度。

### 错误状态

- Provider 错误保留原文和 accepted 版本；
- Edit Scope 错误标出选区和锁定冲突；
- 保存失败显示持续 Banner 和复制全文入口；
- B2 来源版本失效在 Plan Inspector 显示；
- Diff 失败不阻止查看原文和建议稿。

### 关键交互

- 选中文本后出现“重写、扩写、压缩、增强”浮动工具条；
- 生成结果只进入 alternative；
- 接受全部、拒绝全部和按段接受都在 Diff 中完成；
- 段落锁图标固定在正文边缘，锁定文本不可被局部操作覆盖；
- 专注模式隐藏左右侧栏，仅保留场景标题、保存状态和退出按钮；
- 版本恢复创建新版本，不原地覆盖；
- 候选事实和状态必须显式确认，写回 B2 只创建副本。

### 响应式变化

- ≥1280px：260px 树 + 760px 正文 + 320px 检查器；
- 1024–1279px：左侧树收窄或抽屉，正文仍不低于 600px，检查器可收起；
- 768–1023px：正文单列，章节树和检查器分别为左右抽屉；
- <768px：C2.1 设计文本选择、键盘和底部生成面板，本轮不承诺完整移动写作体验。

### 应删除或合并的现有元素

- 删除固定 240px / 主列 / 360px 且无断点的布局；
- 合并顶部所有导入导出和删除按钮到页面菜单；
- 生成面板不再固定占据正文下方；
- 右侧 8 标签合并为四组；
- 段落锁定不再以正文下方长列表呈现；
- 红绿整块 Diff 改为可读的段落级差异。

## 8. 需要用户确认的设计决策

以下每项都有推荐默认值，确认后即可进入 C2.0 实施，不需要再做技术选型：

1. **总体方向**：推荐方向 C“分层密度”，不采用纯营销式宽松或纯高密度工具。
2. **一级导航**：推荐桌面左侧分组导航，平板改抽屉；不保留顶部 9 标签。
3. **项目首页**：推荐新增真正的项目首页，创意输入不再是默认首页。
4. **正文字体**：推荐 UI 全部使用无衬线，正文画布允许用户切换系统宋体阅读模式。
5. **默认密度**：推荐全局 Comfortable，世界书列表、来源、版本、问题和连续性自动使用 Compact。
6. **质量入口**：推荐取消独立“项目质量”一级标签；角色质量回到角色卡，跨模块健康摘要放首页和连续性中心。
7. **设置结构**：推荐新增独立设置页，并把清除数据移入危险区。
8. **色卡文件规范位置**：推荐实施开始时把根目录 `greenboard-palette.jpeg` 复制或移动到 `docs/design-reference/greenboard-palette.jpeg`，文档统一引用该路径；原始色值以 `UI.md` 为准。
9. **参考站约束**：推荐只采用已验证的结构节奏，不采用其绿色品牌配色、大图和轮播。

若用户直接确认推荐方案，后续实施按以上九项默认值执行。
