# 分卷、分章与场景规划方法论

## 职责边界

`VolumePlan` 负责一个宏观阶段的目标、核心冲突、开始/结束状态、预计章节数以及对 B1 Plot Section / Plot Beat 的覆盖范围。它不承载逐场景动作。

`ChapterPlan` 负责一次完整的阅读单元：章节目标、主要冲突、开篇状态、触发、核心转折、结果、结束钩子、视角、状态和信息变化。每章至少应产生一种不可忽略的变化；只重复背景说明的章节会被标记。

`ScenePlan` 负责单一时空和视角下的戏剧动作。场景卡必须能说明入口状态、目标、对抗、行动、转折、结果与出口状态。场景卡是规划，不是小说正文；感官重点和对话功能只记录写作意图。

## 场景入口与出口状态

入口和出口都记录：时间、地点、在场人物、身体状态、情绪状态、当前目标、关系状态、已知信息、持有物品和未解决冲突。按 Volume、Chapter、Scene 顺序排列后，前一场出口是后一场入口的默认基线。

以下变化若没有过渡说明会报告连续性问题：时间或地点不一致；伤势、情绪、目标或关系突然恢复/改变；已知信息消失；持有物品增减；未解决冲突无结果消失。报告不自动修改内容，用户可标记为“确认错误”“有意跳跃”“中间过程省略”或“暂不处理”。第一版只比较显式填写的状态，不推断未填写值，也不计算精确路程。

## 视角规则

支持第一人称、第三人称限知、第三人称全知、多视角和自定义模式。视角配置包含视角角色、是否允许切换、切换标记和自定义规则。

固定检查包括：第三人称限知没有且仅没有一个视角角色；未允许切换却配置多个视角角色；章节视角与场景视角冲突；限知视角引用入口时尚未知的信息。全知与多视角不是错误，但仍需遵守用户配置的切换标记。系统不强制单一写作流派，也不声称能从规划卡判断所有“内心越权”。

## 信息流规则

`InformationItem` 是跨场景稳定的信息实体，分别记录作者是否知情、读者状态、各角色状态、公开/秘密/误导属性以及已验证/未验证状态。`InformationReveal` 记录信息在何章何场、向谁、通过何种方法、基于哪些来源被揭示，并可标记首次揭示。

检查器报告角色提前知情、重复首次揭示、揭示缺少来源和限知视角信息越权。角色误解使用 `misunderstood`，读者误导使用 `misled`，两者不会自动当作事实。关键问题可填写预期回应，未回应仅作启发式提示。

## 铺垫与回收

`ForeshadowItem` 只实现基础生命周期：设置位置、强化位置、计划回收位置、实际回收位置、状态、来源和备注。状态为 `planned/set/reinforced/paid_off/abandoned`。

确定性检查包括回收早于设置、没有设置却回收；启发式检查包括有计划回收但尚无实际回收。废弃项保留但不应继续被新场景引用。同一铺垫的语义重复目前只做文本/ID 层面的提示，不扩展为完整伏笔数据库。

## 章节与场景依赖

Chapter 和 Scene 依赖支持 `causes`、`enables`、`motivates`、`reveals`、`foreshadows`、`resolves`、`contrasts` 和 `continues`。依赖边指向稳定实体 ID；原因、条件和信息揭示应早于消费它的结果。第一版在列表顺序上检查缺失引用、自我依赖、倒序和明显循环，不提供图形画布。

## 节奏检查

Chapter 和 Scene 使用 1–5 的整数记录节奏强度、冲突强度、情绪强度、信息密度与行动密度，并记录预计篇幅。连续三章最高强度、连续三章最低强度、单章场景/信息过载、场景功能高度重复、重大事件缺少余波等都属于启发式判断，而非确定错误。节奏评分用于暴露模式，不是作品质量总分。

## B1 Plot Beat 覆盖

B2 不复制或修改 B1 节点。`PlotBeatCoverage` 为每个 B1 Beat 记录负责完成的 Chapter、负责铺垫的 Chapter/Scene、负责回收或余波的 Chapter/Scene，以及覆盖状态：

- `uncovered`：没有章节负责；
- `planned`：已有章节关联但细节不足；
- `partially_covered`：有完成或铺垫但缺少必要后果；
- `covered`：有明确负责章节并具备所需铺垫/结果；
- `duplicated`：多个章节声明完成同一节点；
- `conflicted`：章节内容或版本与 B1 核心节点冲突。

覆盖是可解释的映射而不是自动排名。B1 节点的标题、原因和结果仍以被引用的 B1 版本为准；切换 B1 版本后，旧 B2 项目保留并显示版本关联风险。

## 锁定、局部生成与版本

Volume、Chapter、Scene 可整体锁定，Chapter/Scene 版本还可以通过 `lockedFields` 锁定单字段。模型只能修改当前生成模式和选定范围；未选模块从原项目保留。生成结果先作为草稿显示，用户确认后才写入本地数据。

Chapter 与 Scene 各自保存版本数组、选中版本、采用版本、父版本、创建原因、采用/废弃状态和差异摘要。替代方案、A3 修订和局部重生成均追加版本，不覆盖旧版。比较报告显示目标、视角、场景顺序、转折、信息揭示、人物/关系变化、B1 偏离、修改成本和连续性风险，不只给单一排名。

## 来源与一致性

所有来源引用必须匹配本次限定上下文中实际包含的来源类型、实体 ID 和版本。被 token 预算排除或版本不匹配的来源标为无效，不能作为有效依据。B2 检查只报告并建议最小修改，不自动改写 B1、A3、角色卡、世界书、章节或场景。
