# 剧情分析方法论

## 目的与边界

分析器用于审查单个剧情方案和最多三个备选分支。它不生成小说大纲、不续写或改写正文，也不会自动修改角色卡、世界书和原始剧情。模型输出是可审查建议，不是事实来源。

## 信息权威等级

| 等级 | 来源 | 处理规则 |
|---|---|---|
| 1 | 用户标记的不可修改设定 | 最高权威；冲突应报告为硬冲突 |
| 2 | 用户当前确认事实、目标和状态 | 不得被低等级内容覆盖 |
| 3 | 已发生剧情 | 视为项目剧情事实 |
| 4 | 角色卡和用户事实型世界书条目 | 作为来源设定引用 |
| 5 | 待分析方案、用户假设和可补充空白 | 作为假设检查，不自动确认 |
| 6 | 已保存的模型推断或建议 | 只能作为参考 |
| 7 | 本次模型推断 | 必须标注推断和置信度 |

事实分类固定为 `confirmed_fact`、`source_setting`、`user_assumption`、`model_inference`、`model_suggestion` 和 `unknown`。低等级信息与高等级信息冲突时，报告冲突来源，不自行选择一个版本。

## 分析维度

1. 因果链：触发、中间步骤、行动与结果、巧合、风险、代价和过度便利。
2. 人物契合度：核心/当前目标、价值观、恐惧、情绪、关系、知识、收益、代价、压力和异常触发。
3. 信息、能力和资源：信息渠道、作者视角越权、能力、时间、资源、权力和机会。
4. 世界观与连续性：规则、身份、年龄、时间、地点、路程、伤势、事件与组织反应。
5. 情感与关系：阶段、触发、信任、行为强度、双方反应、权力关系、余波和铺垫。
6. 戏剧与清晰度：冲突升级、转折、真实风险、节奏和读者理解。

## 问题分类

固定分类为：`hard_contradiction`、`causal_gap`、`motivation_gap`、`character_mismatch`、`information_violation`、`capability_violation`、`world_rule_violation`、`continuity_error`、`relationship_jump`、`tone_mismatch`、`pacing_risk`、`reader_clarity_risk`、`missing_evidence` 和 `opportunity`。

## 严重程度

- `critical`：剧情无法成立，或直接违反核心事实/世界规则。
- `major`：显著破坏人物可信度、信息条件或主要因果链。
- `moderate`：剧情可以成立，但需要明确铺垫、触发或解释。
- `minor`：局部节奏、表达或读者清晰度问题。
- `note`：可选增强机会，不构成错误。

严重程度描述影响大小，与模型置信度相互独立。

## 置信度

- `high`：直接依据用户确认事实或单一明确来源，且合理解释很少。
- `medium`：有来源支持，但存在多个合理解释或缺少部分当前状态。
- `low`：依赖类型惯例、审美判断或关键资料缺失；不得使用绝对语言。

## 评分规则

评分版本为 `analysis-score-v1.0.0`。八个维度均为 0–100 整数：因果完整度、人物动机、人物契合度、世界观一致性、连续性、情感推进、戏剧有效性、读者可理解性。

- 90–100：来源充分，未发现实质缺口。
- 75–89：基本可靠，仅需局部补强。
- 60–74：有条件成立，存在一个或多个中等问题。
- 40–59：需要明显调整。
- 0–39：核心条件不成立或存在硬冲突。

模型可以建议分数，程序会裁剪到 0–100、四舍五入为整数并补齐八维。分数不计算虚假精度；存在 `critical` 时，总体可行性强制显示“当前无法成立”，不得由平均分掩盖。文字结论始终优先于分数。

## 来源引用

引用按来源类型、实体 ID、字段/条目和版本与本次 `AnalysisContext` 清单逐项匹配。只有本次实际包含在上下文且四项均匹配的引用才有效。不存在、被排除或版本不符的引用保留为“无效引用”并显示警告，不能作为有效依据。多来源推断应附全部支持来源，并标记 `inference=true` 和置信度。

## 上下文筛选

用户输入和已发生剧情按权威等级进入上下文；角色卡仅在用户选择时加入。世界书先限制到用户选择的书，再按参与角色、地点、分支文本、条目名称和关键词计算相关性。手动包含优先于自动筛选，手动排除优先于相关性。超出预算时按手动包含、权威等级、相关性顺序裁剪，并在界面列出包含/排除原因和估算 token。

Token 估算采用约两字符一个 token 的保守本地近似，仅用于预算选择，不代表具体模型计费结果。
