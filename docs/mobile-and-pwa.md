# Phase C2.1 移动端、PWA 与跨设备工作区

## 1. 目标与边界

C2.1 保持现有 Next.js Web App 为唯一应用，通过同一套领域模型和页面实现桌面、平板与手机使用。移动端不是功能精简版：A1–C1 的数据仍可查看和编辑，只把次要面板改为抽屉、标签或按需区域。本阶段不包含原生移动应用、第三方云同步、小说文件解析、SillyTavern 扩展或 AI 味诊断。

## 2. 响应式布局规则

断点与验证视口：

| 视口 | 壳层 | 工作区规则 |
| --- | --- | --- |
| 1280px 及以上 | 固定桌面侧栏 | 列表、主区和检查器可并列 |
| 1024–1279px | 56px 紧凑侧栏或抽屉 | 主区优先，检查器可收起 |
| 769–1023px | 抽屉导航 | 单主列，辅助区按标签切换 |
| 360–768px | 顶部上下文栏 + 底部快捷导航 + 抽屉 | 单列；操作栏适配安全区；正文可全屏 |

验收视口固定为 360×800、390×844、412×915、768×1024、1024×768 和 1440×900。所有网格子项必须允许 `min-width: 0`；表格在手机上使用横向滚动容器或卡片视图；按钮、输入、标签和导航的主要触摸目标不小于 44px。安全区使用 `env(safe-area-inset-*)`，页面底部为移动导航和浏览器栏预留空间。

全局应用壳负责一级导航、页面返回、当前项目、联网状态、保存状态和工作区同步状态。各业务页面不得自行复制移动一级导航。手机底部快捷项为首页、小说规划、正文写作和“更多”；“更多”打开完整抽屉，因此所有一级功能仍可触屏到达。

## 3. 一级页面移动策略

### 3.1 C2.1 开始时的移动审计

| 一级页面 | 发现的阻塞与风险 | C2.1 处理 |
| --- | --- | --- |
| 应用外壳 | 1024px 以下只有抽屉，没有手机底部快捷入口；保存状态只表达 localStorage | 增加统一底栏、在线/保存/同步文字状态、焦点返回和安全区 |
| 项目首页 | Hero 字号和四列进度在 360px 下密度偏高 | 缩小标题、进度改两列、快捷卡改两列/单列 |
| 创意输入 | 420px 最小主列与常驻生成检查器造成拥挤 | 手机单列，生成区置后；控件 44px |
| 角色卡 | 三栏编辑、检查器和长字段在小屏同时出现 | 三栏降单列，摘要和检查器按内容顺序呈现 |
| 世界书 | 三栏、一次渲染全部条目、条目标题只支持点击 | 手机单列、每批 30 条、标题改可聚焦按钮并保留上下移动 |
| 剧情分析 | 双栏输入/报告、分支和评分网格过密 | 单列、评分单列、筛选换行、长标签横向滚动 |
| 小说规划 | B1/B2 工具栏和节点操作按钮横向拥挤 | 单列、按钮换行、标签滚动；保留上移/下移 |
| 正文写作 | 导航/正文/生成/检查同时纵向堆叠，键盘后画布过小 | 四视图切换、动态视口全屏、编辑器安全区和触屏选区 |
| 连续性中心 | 15 个标签和知情矩阵表格难以操作，内联固定最小宽度 | 标签滚动、表格语义滚动容器、操作换行 |
| 导入导出 | 角色导入可选文件但缺少统一大小/MIME 校验；没有完整项目入口 | 原生文件选择、25 MiB 校验、完整项目 JSON 备份/恢复 |
| 设置 | 索引与内容双栏；缺少 PWA/工作区/冲突入口 | 手机横向索引，增加 PWA 和安全工作区区段 |

全局还发现弹窗缺少动态视口上限、按钮常低于 44px、Toast 可能被底栏遮挡、`pre` 与长标识符可能撑宽页面。C2.1 通过统一 CSS 约束修复，而不是在每页复制媒体查询。

### 3.2 页面策略

- 创意输入：主表单在前，生成配置折叠；长表单单列，高级约束默认收起。
- 角色卡：角色摘要、编辑器、质量检查按“编辑/检查”切换；字段章节保留，长文本可扩展。
- 世界书：书籍与条目列表先显示，编辑器和检查器按需切换；条目分批渲染；排序始终提供上移/下移按钮。
- 剧情分析：输入和报告单列；上下文、问题和引用分别按标签查看；筛选控件可换行。
- 小说规划：B1/B2 标签可横向滚动；Volume、Chapter、Scene 均使用列表和表单，不依赖拖拽。
- 正文写作：手机只显示“导航、正文、工具、检查”中的一个主视图；正文支持全屏、触屏选区、版本切换、段落锁定与恢复。键盘弹出时编辑器使用动态视口高度。
- 连续性中心：顶部视图标签横向滚动；知情矩阵保留可访问表头并提供滚动容器；长列表分段呈现。
- 导入导出：显式文件选择按钮，校验 JSON MIME/扩展名与大小，用户取消不报错。
- 设置：设置索引转为横向滚动；增加本机/工作区模式、PWA 安装状态、离线和备份说明。

## 4. PWA 实现

Manifest 提供应用名称、短名称、`start_url=/`、`scope=/`、`display=standalone`、主题色、背景色和 192/512/maskable 图标。Service Worker 仅缓存公开应用壳、构建静态资源、图标和离线页：

- 导航使用网络优先，失败时回退已缓存首页或离线页；
- `/_next/static/` 和公开图标使用缓存优先并后台更新；
- `/api/`、非 GET 请求、包含认证头的请求和工作区项目数据从不进入 Cache Storage；
- Provider 请求/响应、API 密钥、令牌、正文项目 JSON 和用户选择的敏感文件不缓存；
- 安装中的新 Service Worker 不主动刷新页面。检测到更新后显示提示，只有用户确认且草稿已保存时才激活并刷新。

离线页解释当前能力并提供返回首页操作。应用壳加载后可离线打开；本机已有项目从 IndexedDB 读取。模型按钮在离线时禁用并说明原因，不建立无限重放队列。

## 5. 本机 Storage Adapter

统一接口以 `WorkspaceProjectRecord` 保存项目 ID、ProjectDraft、版本号和修改时间：

```text
ProjectStorageAdapter
  listProjects()
  readProject(id)
  createProject(record)
  updateProject(id, draft, expectedVersion)
  deleteProject(id, expectedVersion)
  backupProject(id)
  healthCheck()
```

浏览器实现优先使用 IndexedDB 数据库 `story-card-studio`。旧键 `story-card-studio-draft` 仍作为一次性迁移和紧急恢复镜像：首次启动读取并执行 `migrateProjectDraft`，成功写入 IndexedDB 后保留恢复副本，不清空旧值。IndexedDB 不可用或容量不足时回退 localStorage，并显示持久错误和 JSON 导出入口。写入经过短去抖；页面隐藏、冻结和恢复时补做一次保存，避免移动浏览器回收页面造成输入丢失。

## 6. 可选工作区服务器

工作区模式面向用户控制的单用户 Next.js 部署。服务端使用 `WORKSPACE_DATA_DIR` 下的 JSON 记录，不向浏览器暴露文件路径或 Provider 密钥。接口包括健康检查、登录、登出、项目列表、读取、创建、更新、删除和备份。

每次更新携带 `expectedVersion`。版本不相等返回 409、服务端版本与修改时间；客户端停止覆盖，保留本机草稿并可创建“冲突副本”。系统不使用无提示 last-write-wins。服务端写入采用临时文件加原子重命名，避免半写文件。

单用户认证读取 `WORKSPACE_ACCESS_TOKEN` 环境变量。登录成功后创建 HttpOnly、SameSite=Strict、生产环境 Secure 的短时会话 Cookie；会话同时发放仅保存在内存中的 CSRF 值，所有写请求必须携带。服务端校验 Origin/CORS 白名单、限制登录失败频率、限制请求体大小并让会话过期。日志只记录动作、项目 ID、状态码和时间，不记录令牌、Provider 密钥或完整正文。

建议通过 HTTPS 反向代理、受控局域网、Tailscale/WireGuard 或同类私有网络访问。不得把无认证工作区直接暴露到公网。安全 Cookie 在非 localhost 环境要求 HTTPS。

## 7. 离线与同步规则

本机模式离线可打开缓存壳、读取项目、编辑草稿、查看角色卡/世界书/规划/正文并导出。工作区模式离线时先写入本机项目记录并标记 `pending`，不循环发送失败请求。恢复在线后由用户点击同步；自动同步只允许在服务端版本仍等于最后读取版本时执行。

同步状态为 `local`、`saving`、`synced`、`pending`、`offline`、`conflict` 或 `error`。冲突时提供三种安全结果：读取服务端最新版本、把本机内容保存为新项目副本、导出本机 JSON。覆盖服务端必须基于用户重新读取后的新版本再次明确提交。

## 8. 文件、备份与恢复

手机文件能力使用原生文件选择器和浏览器下载/分享能力，不依赖拖拽。JSON 导入上限默认 25 MiB，先检查扩展名/MIME，再解析并运行现有 Schema/迁移。取消选择保持静默；格式、容量和读取错误使用中文说明。项目备份不包含环境变量、Cookie、CSRF、Provider 密钥或调试日志。

建议在重要迁移、跨设备同步和清理项目前导出 JSON。迁移失败时保留原始输入到 `recoveryData`，并保留旧 localStorage 数据以便恢复。

## 9. 可访问性与性能

所有交互具有可见焦点和可读名称，状态同时使用文字与图形，不只靠颜色。抽屉打开后焦点移入，Escape 关闭，关闭后焦点返回触发按钮；背景区域不可交互。错误用 `aria-describedby` 或 `role=alert` 关联。移动端隐藏 hover-only 行为。

大型世界书条目和连续性列表使用分批呈现；正文只渲染当前版本；页面工作区按一级视图懒加载。输入保存去抖，连续性全项目检查不随每次击键运行。超过短任务阈值的操作显示进度和取消入口。

## 10. 已知限制

- 浏览器后台回收、系统极端低存储和无痕模式仍可能限制本机持久化，重要项目应定期导出。
- iOS 的安装入口由 Safari“添加到主屏幕”提供，`beforeinstallprompt` 不一定可用。
- 工作区服务器是单用户文件工作区，不提供多人实时协作、第三方云同步或跨项目权限系统。
- 离线模式不调用真实 Provider，也不缓存失败的模型请求。
- 手机上的大型知情矩阵仍需横向滚动；它保留表格语义，不强制压缩到不可读宽度。
