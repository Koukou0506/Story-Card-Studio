# 作品导入与重建

“作品导入与重建”把用户有权处理的作品文件转换为统一的 Document Source、章节、分块和 Source Span，再由现有人物、世界书、Canon 与文风服务生成草稿或候选。解析器不调用模型，也不直接写项目。

## 支持格式

- TXT：UTF-8、UTF-8 BOM、UTF-16 LE/BE 和 GB18030；低置信度解码需要预览或手动选择编码。
- PDF：优先读取文本层并保留页码。扫描 PDF 进入 OCR 流程，空文本不会当作成功。
- EPUB：校验 ZIP 容器、`container.xml` 和 OPF，按 Spine 阅读顺序处理；Navigation/NCX 仅辅助命名。缺失目录可继续，受保护、损坏或超过安全限制的文件会拒绝。
- DOCX：读取段落、Heading、列表、分页、表格、脚注、尾注、注释、修订和图片引用。可选择最终稿、修订前或保留可判断的修订内容；无法可靠恢复的内容进入警告。
- Markdown：支持 ATX/Setext 标题、Front Matter、引用、列表、图片、链接、HTML、代码块和分隔线。可配置标题映射，以及是否忽略 Front Matter、代码块或内嵌 HTML。

不支持 MOBI，也不分析 EPUB/DOCX 内图片内容。宏、脚本、嵌入附件和外部路径不会执行或跟随。

## 多文件、顺序与版本

一次任务可混合选择以上格式。文件按相对路径和自然数字排序，因此 `第2章`、`chapter-2` 会排在对应的 `第10章`、`chapter-10` 前；用户仍可用上移、下移按钮调整。相对路径可形成卷分组，单文件失败只标记该项，不终止已完成文件。

版本识别比较文件哈希、正规化文本、标题、包含关系和文本相似度，结果只进入候选：完全重复、正规化重复、可能修订、标题冲突、局部重叠或无关。用户可以采用一个、保留全部、标记新旧、合并、排除或暂不处理，系统不会自动删除版本。

## OCR 安装与使用

本机 OCR Adapter 使用 Tesseract CLI；PDF 逐页转图依赖 Poppler 的 `pdftoppm`。两者是可选运行时依赖，缺失时应用仍能构建和使用其他格式，并会显示明确的不可用提示。

建议安装 Tesseract 5、简体中文 `chi_sim`、繁体中文 `chi_tra`、英文 `eng` 语言包，以及 Poppler。OCR 按页保存进度和失败页，可中止、从检查点恢复或只重试失败页。低置信度页必须人工校对；原始 OCR 文本与校对文本分别保存，校对不会覆盖原始结果。页面图像引用、页码、识别区域、置信度和 OCR 版本进入 Source Span。

OCR 不能保证还原复杂多栏、装饰字体、手写内容或低清扫描。校对完成前不应把 OCR 结果当作确认事实。

## 来源追踪

统一 Source Span 可记录文件包相对路径、内容哈希、Parser 版本、EPUB Spine/XHTML、DOCX 段落/Heading/脚注或注释、Markdown 行范围、PDF/OCR 页码与区域，以及字符范围。清洗文本仍保留映射回原始位置的关系。候选资料的“查看原文位置”会使用这些信息；原文件被删除后只保留结构位置和短摘录。

## 项目重建

“创建新的项目数据”从确认的卷章生成项目正文版本，并把人物、世界书、Canon、状态、时间线、剧情线、伏笔和文风资料写成草稿或候选。“补充当前项目”先比较现有章节、正文和资料，为每项给出新增、合并、创建版本、冲突或跳过。

写入前必须生成并确认 Project Rebuild Plan。正文总是创建新的 Draft Version；角色卡和世界书保持草稿；Canon、状态、剧情线、伏笔、Style Profile 与 Language Constraint 保持候选。冲突必须选择保留现有、保留两者或跳过。每项操作记录独立结果，失败任务可以重试，已成功项通过来源计划 ID 避免重复写入，并可从版本历史恢复。

## 隐私、删除与恢复

上传前需要确认处理权利，并明确是否保留原文件、提取文本以及是否允许把所选分块发送给外部模型。纯本地解析和 OCR 不需要模型；Provider 密钥、PDF 密码和完整正文不会写入日志。EPUB/DOCX 解压限制路径、文件数、单项大小、总大小和压缩比，以防路径穿越和 Zip Bomb。

解析任务、文件状态、分块、OCR 页和写入操作均保存检查点。用户可以删除原文件与提取文本但保留结构化候选，也可以删除全部导入数据。跨设备工作区同步结构化项目；未同步的本机原始资产需要在新设备重新选择，才能恢复完整原文预览。

移动端使用系统文件选择器批量选择文件，所有排序、卷章修正、版本审查、OCR 校对和重建确认都提供按钮或分步页面，不依赖拖拽。
