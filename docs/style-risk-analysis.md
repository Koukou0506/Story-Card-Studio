# AI 味与文本机械感诊断

## 功能定位

本模块检查模板化、机械感、过度解释、句式重复、情绪空泛、对话同质和项目文风偏离。它分析的是可解释的文本风格风险，不能可靠证明文本由 AI 或人类创作；风险分数不是作者身份概率，不用于学术作弊或法律判断，也不保证绕过第三方检测器。

## 两层分析

确定性层在本机完成中文分句、段落和句长分布、标点、对话比例、连接词、总结词、副词、重复 N-gram、重复句首和段尾、相邻结构相似度、情绪词、动作及感官词、称谓、对话词汇重合和 Language Constraint 违规。该层不调用模型，离线可用。

模型辅助层只判断统计不可靠的问题，例如动作后的重复解释、过度平滑的情绪推进、对话声音趋同、空泛修辞、模板化转折、冲突过快化解和明显文风偏离。输出必须为结构化 JSON；摘录必须能映射回原文，无法唯一定位时标记为范围不确定。模型失败不会清除确定性报告。

## 基准

- 通用中文小说：使用保守启发式区间。
- 项目文风：读取已确认的 Style Profile 和 Language Constraint。
- 个人样本：用户分别指定文件、题材、章节范围、视角和角色；只保存统计和抽象特征，不保存原文。
- 角色语言：使用角色级语言规则；资料不足时降低置信度。
- 多基准：并列显示基准，不将不同题材或视角样本静默混合。

少于 300 个非空白字符时只给局部提示，不提供稳定总体分数。长文本 API 上限为 500,000 字符；局部优化上限为 100,000 字符。

## 优化、复检与隐私

局部优化复用 B3 Edit Scope、Draft Version、Revision 和段落 Diff。选区之外文本与锁定段落受既有校验保护；建议稿默认是 alternative，用户可全部、按段接受或拒绝。复检比较重复率、句长、段落、情绪/动作密度、对话区分度和语言规则违规，但指标下降不等于创作质量必然提高。

用户可选择纯本地诊断。只有明确启用模型辅助时才发送当前范围和必要指标；不发送无关项目内容，不在日志中记录全文或 Provider 密钥。个人样本基准可以删除，删除原文后抽象统计可独立保留。

## SillyTavern

Extension 面板提供“诊断选定文本的 AI 味与机械感”，支持最近消息、手动范围、仅角色回复和粘贴文本，并在发送前预览。结果显示风险摘要，可导出或在独立 APP 查看完整报告。MVP 不修改 SillyTavern 历史聊天。

## 已知限制

确定性词表和结构相似度属于中文小说启发式；方言、实验性写法、诗化文本和极短文本可能误报。模型判断仍可能遗漏或误解语境。个人样本应由用户按题材、视角和角色分组，当前不会自动完成跨题材聚类。
