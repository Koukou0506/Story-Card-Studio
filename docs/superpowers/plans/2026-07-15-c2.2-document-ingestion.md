# Phase C2.2 Document Ingestion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development and execute each task in order. Each production behavior must first be demonstrated by a failing Vitest test.

**Goal:** Add safe TXT/PDF ingestion, traceable reverse modelling, candidate review, and explicit integration with A1/A2/B3/C1 without overwriting confirmed project data.

**Architecture:** A versioned `DocumentIngestionProject` stores structured metadata, chapters, chunks, jobs, candidates, and asset references. Parser, normalizer, segmenter, chunker, resolver, statistics, converters, and orchestrator remain pure services; original bytes and long text are stored through a document asset adapter instead of ordinary logs or repeated ProjectDraft autosaves.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript strict, Zod 4, Vitest, browser `TextDecoder`, PDF.js (`pdfjs-dist`), IndexedDB.

## Global Constraints

- Preserve all A1–C2.1 behavior and increment `ProjectDraft` to v8 through a recovery-safe migration.
- Accept only TXT and PDF; do not add DOCX/EPUB/MOBI or OCR implementation.
- Every extracted result remains `candidate` or `draft` until an explicit user action.
- Never cache, log, or return Provider keys, PDF passwords, full request text, or full novel content.
- Do not send a whole novel to a Provider; only selected chunks with visible source spans.
- Do not implement SillyTavern extensions/plugins or AI-authorship diagnosis.
