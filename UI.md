# Phase C2.0：视觉系统与全局 UI 重构

## 当前阶段

当前阶段为 Phase C2.0。

本阶段先于 Phase C2.1 移动端适配执行。

目标是重新设计现有应用的视觉系统和主要界面结构，使应用形成统一、现代、具有编辑感的创作工作台体验。

参考来源：

- 视觉结构参考：用户提供的 Greenboard 页面；
- 配色参考：docs/design-reference/greenboard-palette.png；
- 当前应用功能和信息架构；
- 用户确认的设计规范。

参考网站仅用于提取：

- 留白方式；
- 字体层级；
- 大型块面；
- 圆角语言；
- 导航密度；
- 信息呈现节奏。

不得复制：

- Greenboard 商标；
- Logo；
- 文案；
- 插图；
- 波形资产；
- 专有图标；
- 页面逐像素布局；
- 其他品牌识别元素。

## 视觉定位

应用采用 Editorial Creative Workspace 风格。

核心特征：

- 暖米白画布；
- 深棕文字；
- 灰橄榄辅助色；
- 钴蓝主交互色；
- 大标题；
- 宽松页面留白；
- 大圆角块面；
- 克制边框；
- 极少阴影；
- 清晰的信息层级；
- 编辑器和正文内容优先。

## 设计 Token

颜色：

- canvas: #FCFCF1
- surface: #F4F4E0
- surface-muted: #C4C4AF
- surface-cool: #C2C4BD
- olive: #81816C
- olive-soft: #999882
- neutral: #9A9A90
- neutral-dark: #7F7E76
- ink: #1E1412
- accent: #294A97
- white: #FFFFFF

正文必须使用 ink 或满足可访问性对比度的颜色。

olive、olive-soft 和 neutral 不得用于小字号主要正文。

accent 仅用于：

- 主操作；
- 当前选中状态；
- 链接；
- 焦点；
- 少量关键数据。

不得引入不必要的高饱和色或大面积渐变。

## 重构原则

- 不修改业务逻辑；
- 不修改领域模型语义；
- 不改变导入导出格式；
- 不删除已有功能；
- 不以视觉重构为理由大规模重写数据层；
- 优先建立设计 Token 和共享组件；
- 页面不得各自硬编码颜色、圆角和阴影；
- 保留现有测试，并增加视觉和交互回归测试；
- C2.0 建立响应式基础，但完整移动端体验留到 C2.1；
- 不在设计方案批准前批量修改页面。

## 执行顺序

1. 审计当前 UI；
2. 分析参考网页和色卡；
3. 建立设计原则；
4. 输出设计 Token；
5. 输出关键页面线框图；
6. 输出三个代表页面的高保真方案；
7. 用户确认；
8. 建立共享 Design System；
9. 重构应用外壳；
10. 按优先级迁移页面；
11. 运行视觉和功能回归；
12. 清理旧样式。

## 优先页面

第一批：

1. 应用外壳和导航；
2. 项目首页；
3. 角色卡；
4. 世界书；
5. 正文写作。

第二批：

1. 剧情分析；
2. 小说规划；
3. 连续性中心；
4. 导入导出；
5. 设置。

不得在第一批设计未确认前重构全部页面。
